>

## 9. UI 出图规范

> UI 设计阶段（`开发流程SOP.md` §四 阶段 2）怎么产出**可开发**的设计稿。**开工先和用户选路线。**

**三条路线终点统一**——都是「可开发的高保真静态稿」；区别只在**怎么得到它**。开工时按项目情况选一条：

- **路线 A · 直接高保真**
  - 做法：逐屏直接出可开发的高保真静态视觉稿。
  - 适合：风格已经清晰、能一次写对（多见于功能迭代、已有设计规范可执行）。
  - 产物：高保真静态稿 → 直接开发。

- **路线 B · SVG 线稿先行**
  - 做法：① 逐屏出低保真 SVG 线稿、定结构布局 → ② 据线稿逐屏转成可开发的高保真静态稿。
  - 适合：信息架构复杂、想先定结构再谈视觉（多见于新产品首个核心页面）。
  - 产物：线稿（中间产物，可丢）+ 高保真静态稿 → 开发。

- **路线 C · AI 生图作参考**
  - 做法：① 用 AI 生图工具生成视觉概念图作参考（**先问用户要不要生图、有没有生图工具，没有就跳到 A 或 B**）→ ② **先结构化分析参考图**（布局栅格 / 组件 / 间距 / 层级 / 配色逐项拆解，有 `image-to-code` 类工具就用它分析）→ 据分析重写成可开发的高保真静态稿。
  - 适合：视觉风格没思路、需要灵感来源（多见于新产品起步定调阶段）。
  - 产物：AI 生图（参考，不可作开发稿）+ 高保真静态稿 → 开发。
  - **铁律**：① AI 生图细节不准、组件不通用、像素对不上，**绝不能直接作为开发稿**，必须再转一道可开发的高保真稿才算完成（防拿生图交差）。② 转写**必须基于对参考图的结构化分析、不能凭感觉临摹**——否则成品和生图除了配色基本是两个东西（布局、登录卡结构、关键卡片全走样）。

**三条路线共同的铁律（不可破）：**

- **一画面一稿**：每个页面 / 关键状态单独一张稿，**绝不把多个页面塞进一张**。
- **真实视口尺寸**：按目标设备的真实视口出图（断点见 §6），不出缩略图、不缩排成一堆小图。
- **可开发**：稿子要能让人照着 1:1 实现；糊到看不清细节 = 没出。

**产出放哪**：线稿 / 参考图 / 视觉稿都放对应功能的 `资源/`；新增的颜色 / 字体 / 组件回填本规范 §2–§5。
