## 9. UI 出图规范

> UI 设计阶段（`开发流程SOP.md` 阶段 3）怎么产出**可开发**的设计稿。**开工先和用户选路线。**

**两条路线，开工时按项目选**（简单 / 结构已清晰走 A，复杂 / 想先定结构走 B）：

- **路线 A · 直接高保真**：逐屏直接出高保真静态视觉稿。
- **路线 B · 线框先行**：① 逐屏出低保真线框图、定结构布局 → ②【可选】用 AI 生成视觉概念图作参考（**先问用户要不要生图，没有生图工具就跳过**）→ ③ 据线框 / 参考图逐屏转成高保真静态视觉稿。

**两条路线共同的铁律（不可破）：**

- **一画面一稿**：每个页面 / 关键状态单独一张稿，**绝不把多个页面塞进一张**。
- **真实视口尺寸**：按目标设备的真实视口出图（断点见 §6），不出缩略图、不缩排成一堆小图。
- **可开发**：稿子要能让人照着 1:1 实现；糊到看不清细节 = 没出。

**产出放哪**：线框 / 视觉稿放对应功能的 `资源/`；新增的颜色 / 字体 / 组件回填本规范 §2–§5。
