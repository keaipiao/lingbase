# 设计规范

> 项目级前端设计系统——**所有功能的 UI 都引用这里，不各自造**。纯后端 / 无前端的项目可整份删除本文件（保留则首行标「本项目无前端，不适用」）。
>
> **整份规范边做边长**，按一条朴素标准定先后：**改一次等于全站返工的先定，改起来便宜、预先列多半用不上的边做边长。**
> - **先定**（§1 基调 / §2 主色 / §3 字体）：换字体 / 换主色 = 全站重排，所以**做第一个界面时就拍板**。
> - **边做边长**（§4 起 间距 / 组件 / 响应式 / 动效 / 文案…）：遇到一个记一个，别预先穷举。
>
> 已确定的就写、未定的留空待补；章节固定、按编号保留，不适用就注明「不适用」、不删。
