# 架构总览

> 项目级技术地图——给 AI 和接手者快速建立全局认知。
> **living document，随架构演进持续增量更新**，永远停在当前真实状态：每章节**已确定的就写、未确定的留空待补**——既不写未定 / 假设内容（见 `CLAUDE.md`「文档只写已确定结论」），也别因「以后还会补」就把已能确定的章节空着。
> 单个功能的内部设计在各功能 `设计.md`；这里只放**跨功能的全局架构**。
> 章节固定、按编号保留，不适用就注明「不适用」、不删。
