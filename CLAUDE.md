# CLAUDE.md

> 本文件是 AI 编码工具进入**本项目**的总入口。每次对话开工前先读。
> 全程**用简洁的中文回复**。

---

## 0. 最高优先级（硬性规定）

1. **流程唯一**：本项目的开发流程、计划拆分、进度记录，一律按 `docs/开发流程SOP.md` + 灵计一动看板执行。**无论你（AI）自带或启用了什么工作流 skill，都必须服从本规范**，不得用 skill 自己的 todo / 任务清单 / 流程替代。
2. **新功能先看 SOP**：开发新功能 / 新里程碑，先读 `docs/开发流程SOP.md`，严格按它的阶段（需求沟通 → 立项 → 调研 → UI → 技术设计 → 实现 → 测试 → 发布 → 收尾）走，逐阶段停下等用户确认。
3. **先需求、再复制模板**：**先做需求沟通、确认值得做**，才从复制 `docs/功能/_模板/` 开始建功能目录（SOP 阶段 0 → 1）。不跳需求直接建目录，也不自创结构、不另写一套文档。
4. **不轻信记忆、核实再答**：涉及版本号、API、库 / 框架 / 工具 / 模型的最新状态、定价、命令用法时，**先联网查官方 / 最新来源核实当前事实**再下结论 —— 训练记忆、甚至旧的搜索结果都可能滞后（例：某 AI 模型早已迭代到新版，仍照旧推荐就会把用户带进坑）。核实后**附来源**；查不到或拿不准就**明说「未核实」**，不冒充确定。

## 1. 进度看板：灵计一动（开箱即用）

本项目用 **[灵计一动](https://lingji1dong.com)** 做进度看板（配套 MCP，让 AI 直接读写）：**计划、进度、过程笔记都在看板，不写进文档**。

- **会话开始 / 动手前**：读根目录 `.lingji` 拿 project id（形如 `{"project":N}`，没有就 `create_project` 新建并写回 `.lingji`）；调一次 `get_project_context` 拿项目专属约束，整会话牢记。
- **判断**：纯问答 / 看代码 → 直接答、不建计划；要动手做事 → 往下走。
- **记灵感**：`create_inspiration` 把需求按【**用户原话**】记成灵感（不改写 / 润色 / 翻译）。
- **拆计划**：先和用户聊清需求再拆。`get_plan` 看有无大纲；没有就 `set_plan` —— 缩进列表 ≤3 层，每个叶子是「一次能做完、一眼能看懂」的具体步骤，用「做成什么」描述，**不写行号 / 类名 / 方法名 / 字段名 / 文件名**等代码级细节。**首次建好整份计划 → 发给用户确认或调整再开工**。
- **边做边回填**：每个叶子 动手前 `update_node_status(doing)`、做完 `done`；只勾叶子（父节点自动汇总）。返回带「请先 get_plan 同步」→ 立刻重新 `get_plan`（用户在网页改了计划）。
- **边做边记笔记**：实现 / 决策 / 踩坑 / 验证用 `add_note` 给对应叶子追加（多条成日志、不覆盖）。
- **改计划**：还没做的（undone）节点直接 `rename` / `move` / `delete`；已完成 / 进行中的**别直接改删**，用 `add_node` 加「请把 X 改成 Y / 删除 Z」说明节点（保留历史）。

> 没配置灵计一动 MCP 时，先按 [lingji1dong.com](https://lingji1dong.com) 接入，再开工。

## 3. 编码红线

- **中文优先**：用中文回复。
- **代码自解释**：靠命名和结构让代码自己说清「做了什么、怎么做」——这些不写注释。注释只写**代码表达不出来的东西**：这段代码的意图 / 解决什么问题、为什么这么选（而非别的写法）、隐藏的坑与约束、必要的外部背景（关联 issue / 规格）。注释用中文。
- **先明确再动手**：需求不清先问用户，不在不清楚时就改；明示假设，多解读先列出不默选；卡住停下来问，不硬猜。
- **外科手术式修改**：只动与需求直接相关的代码。不顺手「改进」无关代码 / 注释 / 格式，不重构没坏的东西。
- **最小规格、简洁优先**：能简不繁，不做未要求的抽象 / 灵活性 / 防御性。审视产出：「资深工程师会说这过度设计了吗？」是 → 重写。
  - **配置最小化**：只加**现在真正必要**的环境变量；**feature flag / 灰度开关默认不加**（除非确有运行时切换或灰度上线的硬需求）；环境差异（如验证码 dev mock vs prod 邮件）用 **dev/prod profile** 区分，**不为此加 `.env` 开关**；不留废弃 / 占位变量。
- **一致性**：改任一处，检查上下游所有代码 / 文档 / 页面一并改。文档只写**已确定的结论**，不留「待定 / 不确定 / 假设」。
- **阶段闸门**：每个阶段产出后**停下**，等用户明确「通过 / 继续」再进下一阶段；沉默 ≠ 同意（用户说「一口气做完」才放开）。
